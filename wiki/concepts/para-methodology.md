# PARA Methodology

**Type:** Concept  
**Origin:** Tiago Forte — Forte Labs / Building a Second Brain

## Summary

PARA is a universal information organization framework built around **actionability**, not topic. Everything you encounter fits into one of four mutually exclusive categories, kept deliberately shallow (max 4 levels deep).

## Core Principle

Organize by **when you'll need it**, not **what it is**. A note about nutrition in a marathon training project goes in `Projects/Marathon` — not `Resources/Nutrition`. Move it to Resources only when the project ends.

## The Four Buckets

- **Projects** — has a goal + deadline; active work
- **Areas** — ongoing responsibility with a standard (health, finances, relationships)
- **Resources** — interest or reference with no deadline
- **Archives** — completed/inactive items from any of the above

## Why It Works

- Shallow hierarchy → always know where to look
- Actionability-based → mirrors how you actually use information
- Universal → same structure across notes, files, cloud storage, tasks
- Progressive Summarization → notes improve with each interaction
