## Core Insight

The Ralph Loop (autonomous AI coding loop driven by a PRD) can be transformed from a reward-maximizing task-completer into a MOP-driven cognitive explorer when combined with the Maximum Occupancy Principle and Theory of Mind. The critical addition is treating epistemic energy (context window, token budget, confidence) as the energy reservoir that forces goal-directed behavior near depletion.

## Key Ideas

**Ralph Loop as reward maximizer:** A standard Ralph Loop reads a PRD, checks a progress file, implements one feature, tests, commits, repeats. It collapses to deterministic behavior — shortest path to task completion.

**MOP transformation:** An MOP-driven loop doesn't just want to finish tasks — it wants to map the conceptual space of the research. It treats surviving (maintaining viable context) as the prerequisite for continued exploration, not the goal.

**Theory of Mind integration:** Scientific literature synthesis requires second-order reasoning — understanding nested beliefs like "Researcher A asserts that Model B fails because it assumes Condition C." This creates the serial processing bottleneck identified in developmental psychology.

**Epistemic calibration as absorbing state avoidance:** The conversation identifies that MOP's exploration drive must be counterbalanced by strict epistemic filters to prevent hallucinated scientific claims. This maps to MOP's absorbing states: unverified connections are treated as epistemic dead-ends.

## Nuances for Research Application

1. **Context window chokehold:** Each loop iteration wipes memory; use RAG/MCP to dynamically pull needed content
2. **Testing concepts vs. code:** Scientific claims need symbolic logic verification, not just error-free execution
3. **Epistemic drift:** Continuous synthesis generates hallucinated correlations; Type II error avoidance is preferred over Type I
4. **PDF conversion quality:** Preprocessing pipeline needed before loop ingestion
